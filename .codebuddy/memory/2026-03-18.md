## LittleFS 替代 SD 卡存档改造

将 ESP32-S3 的游戏存档系统从 SD 卡改为 LittleFS（内部 Flash），同时保留 SD 卡硬件代码供未来扩展。

### 设计架构
- **`saveFS`** (fs::FS&) — 统一文件系统抽象，指向 LittleFS 或 SD
- **`saveReady`** — 存档系统是否就绪
- **`sdReady`** — SD 硬件是否就绪（保留供未来 asset 加载）
- **`USE_LITTLEFS_SAVE`** 宏 — ESP32-S3 默认 =1 (LittleFS)，其他板 =0 (SD)

### 修改内容
1. **JurassicLifeFR.ino**:
   - 新增 `USE_LITTLEFS_SAVE` 配置宏（第18-26行）
   - 新增 `#include <LittleFS.h>` 条件编译
   - 新增 `saveFS`/`saveReady` 统一抽象
   - 新增 `lfsInit()` + `storageInit()` 初始化函数
   - 所有文件操作 SD.xxx → saveFS.xxx
   - 所有存档判断 sdReady → saveReady
   - SD 硬件代码全部保留，sdInit() 仍会被调用（即使 LittleFS 模式）
2. **README.md**:
   - 第9节更新为「LittleFS / SD card」双模说明
   - 硬件需求中 SD 卡标注为未来扩展用
   - 库依赖添加 LittleFS
